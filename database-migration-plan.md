# 數據庫遷移計劃

## 問題分析

### 當前問題
1. **數據丟失**: 每次部署都會清空 SQLite 數據庫
2. **無持久化**: 用戶數據、查詢記錄、投資組合都會丟失
3. **擴展性差**: SQLite 不適合多實例部署
4. **備份困難**: 本地文件難以備份和恢復

## 解決方案選項

### 選項 1: PostgreSQL (推薦)
**優點:**
- 生產級數據庫，穩定可靠
- 支持複雜查詢和事務
- 良好的擴展性和性能
- 豐富的數據類型支持
- 強大的備份和恢復功能

**適用場景:** 生產環境，需要高可靠性和性能

### 選項 2: MySQL
**優點:**
- 廣泛使用，文檔豐富
- 性能優秀
- 成本較低
- 易於管理

**適用場景:** 中等規模應用

### 選項 3: 雲數據庫服務
**優點:**
- 無需維護服務器
- 自動備份和恢復
- 高可用性
- 彈性擴展

**推薦服務:**
- AWS RDS (PostgreSQL/MySQL)
- Google Cloud SQL
- Azure Database
- PlanetScale (MySQL)
- Supabase (PostgreSQL)

## 推薦方案: Supabase PostgreSQL

### 為什麼選擇 Supabase
1. **免費額度**: 500MB 數據庫，足夠初期使用
2. **PostgreSQL**: 功能強大，性能優秀
3. **實時功能**: 支持實時數據同步
4. **REST API**: 自動生成 REST API
5. **認證系統**: 內建用戶認證
6. **管理界面**: 直觀的數據庫管理界面
7. **備份恢復**: 自動備份和時間點恢復

### 實施步驟

#### 1. 創建 Supabase 項目
- 訪問 https://supabase.com
- 創建新項目
- 獲取數據庫連接字符串

#### 2. 數據庫結構遷移
- 將 SQLite 表結構轉換為 PostgreSQL
- 創建相同的表和索引
- 設置外鍵約束

#### 3. 數據遷移
- 導出現有 SQLite 數據
- 轉換數據格式
- 導入到 PostgreSQL

#### 4. 代碼修改
- 更換數據庫連接庫
- 修改 SQL 查詢語法
- 更新環境變量

#### 5. 部署配置
- 設置數據庫連接字符串
- 配置環境變量
- 測試連接
