# 如何測試真實查詢記錄

## 🎯 問題說明

查詢記錄管理頁面顯示的是測試數據，而不是真實的用戶查詢記錄。這是因為：

1. **用戶需要登入** - 查詢記錄API需要認證
2. **需要進行真實預測** - 只有進行股票預測時才會記錄
3. **需要包含credentials** - fetch請求必須包含cookies

## ✅ 已修復的問題

1. **添加了credentials: 'include'** - 所有查詢記錄API請求現在都包含cookies
2. **完整的查詢記錄功能** - `logStockQuery()`, `logAIQuery()`, `logUserQuery()`
3. **自動記錄** - 每次預測完成後自動記錄

## 🚀 測試步驟

### 步驟1: 登入系統

1. 打開瀏覽器訪問你的應用
2. 訪問登入頁面: `http://your-domain/login.html`
3. 使用admin帳戶登入:
   - 用戶名: `admin`
   - 密碼: `admin123`
4. 確認登入成功 (應該重定向到主頁)

### 步驟2: 進行股票預測

1. 在預測器頁面輸入股票代碼 (如 `AAPL`)
2. 選擇預測期間 (如 `30天`)
3. 點擊"運行預測"按鈕
4. 等待預測完成

### 步驟3: 檢查控制台

1. 打開瀏覽器開發者工具 (F12)
2. 切換到 Console 標籤
3. 查找以下消息:
   - ✅ `股票查詢已記錄: Query logged successfully`
   - 或 ⚠️ 錯誤消息

### 步驟4: 查看查詢記錄

1. 訪問查詢記錄管理頁面: `http://your-domain/admin-queries.html`
2. 刷新頁面
3. 應該看到新的真實查詢記錄

## 🔍 常見問題

### Q1: 控制台顯示 "⚠️ 股票查詢記錄失敗: 401"

**原因**: 用戶沒有登入或session過期

**解決方案**:
1. 重新登入
2. 確保使用正確的用戶名和密碼
3. 檢查cookie是否被瀏覽器阻止

### Q2: 控制台顯示 "⚠️ 股票查詢記錄失敗: 503"

**原因**: 數據庫未初始化

**解決方案**:
1. 檢查服務器日誌
2. 確保Supabase連接正常
3. 運行 `node check-database-status.js` 檢查數據庫狀態

### Q3: 沒有看到任何控制台消息

**原因**: 查詢記錄功能可能沒有被觸發

**解決方案**:
1. 確保使用的是最新版本的前端代碼
2. 清除瀏覽器緩存
3. 強制刷新頁面 (Ctrl+Shift+R)

### Q4: 記錄仍然是測試數據

**原因**: 可能顯示的是舊的測試數據

**解決方案**:
1. 清除測試數據: 運行 `node simulate-real-query.js`
2. 或在Supabase Dashboard中手動刪除測試記錄
3. 重新進行股票預測

1. **控制台消息**:


2. **查詢記錄管理頁面**:
   - 總查詢數增加
   - 新的真實查詢記錄出現在列表中
   - 記錄包含正確的股票代碼、預測結果、時間等

3. **記錄內容**:

   - 類型: stock
   - 用戶: admin

## 💡 提示

1. **首次使用**: 第一次進行預測後，可能需要刷新查詢記錄管理頁面
2. **多用戶測試**: 可以創建多個用戶並測試各自的查詢記錄
3. **數據清理**: 定期清理測試數據以保持數據庫整潔
4. **性能監控**: 查看查詢記錄的響應時間，確保性能良好
