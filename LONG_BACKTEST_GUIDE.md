# 📊 長時間回測功能指南

## 🎯 功能概述

現在支持最長 **5年（1825天）** 的回測分析，讓您能夠對股票預測模型進行更全面的歷史驗證。

## ⚡ 性能優化

### 🔧 智能算法優化
- **短期回測（≤1年）**：使用每日步長，提供最精細的分析
- **長期回測（>1年）**：自動切換到每週步長，大幅提升性能
- **數據緩衝**：為長時間回測預留額外的歷史數據緩衝

### 📈 回測範圍選項
| 時間範圍 | 天數 | 步長 | 預計處理時間 |
|---------|------|------|-------------|
| 30天 | 30 | 每日 | < 10秒 |
| 90天 | 90 | 每日 | < 30秒 |
| 6個月 | 180 | 每日 | < 1分鐘 |
| 1年 | 365 | 每日 | < 2分鐘 |
| 2年 | 730 | 每週 | < 3分鐘 |
| 3年 | 1095 | 每週 | < 4分鐘 |
| 4年 | 1460 | 每週 | < 5分鐘 |
| 5年 | 1825 | 每週 | < 6分鐘 |

## 🚀 使用方法

### 1. 在主應用中使用
1. 填寫股票代號（如 AAPL）
2. 點擊「運行回測分析」按鈕
3. 在彈出對話框中輸入回測天數（30-1825天）
4. 等待分析完成

### 2. 使用測試頁面
訪問 `http://localhost:3000/test-backtest.html` 進行專門測試：
- 選擇不同的時間範圍
- 查看詳細的進度信息
- 獲得完整的調試信息

## 📋 數據源支持

### 優先級順序
1. **FMP (Financial Modeling Prep)** - 最可靠，支持完整歷史數據
2. **Finnhub** - 高質量數據，良好API限制
3. **Polygon.io** - 專業級數據，適合大規模分析
4. **Alpha Vantage** - 備用數據源

### 歷史數據要求
- **短期回測**：需要至少 30-100 天的歷史數據
- **長期回測**：自動獲取最多 2000 天的歷史數據
- **數據驗證**：自動檢查數據完整性

## 🔍 分析指標

### 核心性能指標
- **總交易次數**：回測期間的總交易數量
- **勝率**：正確預測方向的百分比
- **總回報**：累積投資回報率
- **夏普比率**：風險調整後的回報率
- **最大回撤**：最大連續虧損幅度

### 預測準確度指標
- **MAE (平均絕對誤差)**：預測價格與實際價格的平均偏差
- **RMSE (均方根誤差)**：預測誤差的標準差
- **方向準確度**：預測價格方向的準確性

## ⚠️ 注意事項

### 系統要求
- **網絡連接**：長時間回測需要穩定的網絡連接
- **瀏覽器**：建議使用現代瀏覽器（Chrome, Firefox, Safari, Edge）
- **API限制**：某些數據源可能有API調用限制

### 最佳實踐
1. **首次使用**：建議先測試短期回測（30-90天）
2. **長時間回測**：確保網絡穩定，避免中途中斷
3. **數據驗證**：檢查回測結果的合理性
4. **多模型比較**：同時運行多個模型進行比較

## 🛠️ 故障排除

### 常見問題
1. **數據不足**：某些股票可能沒有足夠的歷史數據
2. **API限制**：數據源可能達到調用限制
3. **網絡超時**：長時間回測可能因網絡問題中斷

### 解決方案
1. **檢查API密鑰**：確保所有數據源API密鑰有效
2. **重試機制**：失敗時可以重新嘗試
3. **分段測試**：將長時間回測分解為多個短時間段

## 📊 結果解讀

### 勝率分析
- **>70%**：優秀的預測能力
- **60-70%**：良好的預測能力
- **50-60%**：一般的預測能力
- **<50%**：需要改進的預測能力

### 回報分析
- **正回報**：模型在歷史數據上表現良好
- **負回報**：模型可能需要調整或該股票不適合此策略

### 風險評估
- **低回撤**：模型風險控制良好
- **高回撤**：需要注意風險管理
